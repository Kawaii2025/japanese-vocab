# 使用指南

## 快速开始

### 方式一：使用Vue 3版本（默认推荐）

1. 安装依赖：
```bash
npm install
```

2. 启动开发服务器：
```bash
npm run dev
```

3. 在浏览器中访问 `http://localhost:3000`

**优点：** 代码结构清晰，便于二次开发和维护

### 方式二：使用原始HTML版本（无需安装）

1. 直接在浏览器中打开 `index-original.html` 文件
2. 开始使用所有功能

**优点：** 无需任何安装，即开即用

## 功能使用说明

### 1. 输入单词

支持两种格式：

#### 文本格式
```
中文意思,日语原文,纯假名
```
例如：
```
你好,こんにちは,こんにちは
谢谢,ありがとう,ありがとう
```

#### Markdown表格格式
```markdown
| 日文 | 中文 | 假名 |
| ---- | ---- | ---- |
| こんにちは | 你好 | こんにちは |
| ありがとう | 谢谢 | ありがとう |
```

### 2. 练习单词

1. 点击"生成练习表格"按钮
2. 在"练习（输入假名）"列输入假名
3. 点击✓按钮检查答案
4. 查看正确/错误反馈

### 3. 显示控制

#### 全局控制
- **显示/隐藏假名**：一键隐藏或显示所有假名
- **显示/隐藏单词**：一键隐藏或显示所有日语原文

#### 单行控制
- 👁️ 图标：单独控制某一行的显示/隐藏
- 点击后自动记录为"不熟悉"

### 4. 错题管理

- 自动记录所有错误答案
- 显示答案差异对比
- 标记已纠正的错题
- 支持导出为Markdown格式

### 5. 不熟悉单词

- 自动记录点击显示的单词
- 集中复习功能
- 支持清空记录

### 6. 导出功能

#### 导出未完成
导出所有未练习的单词为Markdown表格

#### 合并导出
导出所有不熟悉和错题（自动去重）

### 7. 其他功能

- **朗读**：🔊 按钮朗读日语假名
- **打乱顺序**：随机打乱单词顺序
- **重新编辑**：双击或点击编辑按钮重新作答
- **清空**：清空所有内容重新开始

## 快捷键

- `Ctrl + Enter` (或 `Cmd + Enter` on Mac)：提交输入，生成表格

## 统计信息

实时显示：
- 总单词数
- 已练习数量
- 正确率
- 总错误数
- 不熟悉数量

## 常见问题

### Q: 如何重新作答某个单词？
A: 双击输入框或点击"编辑"按钮（铅笔图标）

### Q: 如何导出错题？
A: 点击"复制为Markdown"按钮，会自动复制到剪贴板

### Q: 表格格式识别失败怎么办？
A: 检查表格格式是否正确，确保有分隔线 `| ---- |`

### Q: 朗读功能不工作？
A: 确保浏览器支持Web Speech API，建议使用Chrome浏览器

### Q: 可以保存进度吗？
A: 当前版本不支持持久化存储，刷新页面会清空数据

## 技巧建议

1. **先隐藏假名练习**：点击"隐藏假名"按钮，测试记忆
2. **使用打乱功能**：避免记忆顺序而非内容
3. **重点复习错题**：查看错题表，了解薄弱环节
4. **导出未完成**：每次练习后导出未掌握的单词
5. **利用朗读功能**：加强听力记忆

## Vue 3 版本开发

### 修改组件
所有组件位于 `src/components/` 目录：
- `HeaderComponent.vue` - 页面标题
- `VocabInputComponent.vue` - 输入区域
- `VocabTableComponent.vue` - 练习表格
- `UnfamiliarWordsComponent.vue` - 不熟悉单词
- `MistakesTableComponent.vue` - 错题记录
- `StatsComponent.vue` - 统计信息

### 修改状态逻辑
编辑 `src/composables/useVocabulary.js`

### 修改工具函数
编辑 `src/utils/` 目录下的文件

### 添加新功能
1. 创建新组件或修改现有组件
2. 在 `App.vue` 中引入和使用
3. 使用 `useVocabulary` composable 管理状态
